# Internationalization (i18n)

The project uses `next-intl` for internationalization.

## Configuration

- **Config File**: `src/shared/config/i18n/request.ts`
- **Routing**: `src/shared/config/i18n/routing.ts`
- **Middleware**: `src/proxy.ts` (proxies requests to next-intl middleware)

## Supported Locales

- `en` (English)
- `pt` (Portuguese - Default)
- `es` (Spanish)

## Adding Translations

### 1. Add keys to JSON files
Update the message files in the root `messages/` directory:
- `messages/en.json`
- `messages/pt.json`
- `messages/es.json`

Structure keys by feature:

```json
{
  "HomePage": {
    "title": "Welcome"
  },
  "Blog": {
    "readMore": "Read more"
  }
}
```

### 2. Usage in Components

```tsx
import { useTranslations } from 'next-intl'

export function MyComponent() {
  const t = useTranslations('HomePage')
  
  return <h1>{t('title')}</h1>
}
```

### 3. Usage in Async Server Components

```tsx
import { getTranslations } from 'next-intl/server'

export async function Page() {
  const t = await getTranslations('HomePage')
  
  return <h1>{t('title')}</h1>
}
```

## Adding a New Language

1. Add the locale code to `src/shared/config/i18n/routing.ts`.
2. Create a new JSON file in `messages/` (e.g., `fr.json`).
3. Update `src/app/[locale]/layout.tsx` metadata (if specific localized metadata is needed).
